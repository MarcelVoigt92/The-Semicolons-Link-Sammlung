HTML5

Obwohl einer der Gründe für den Sieg des "Internet Explorers" über "Netscape" seine bessere Implementierung von CSS und der HTML-Standards war, stagnierte bei Microsoft die Weiterentwicklung von HTML, während das W3C an XHTML2 weiterarbeitete, ohne die Sprache um neue Features erweitern zu wollen.

Allerdings begannen Browserhersteller unter dem Schlagwort HTML5 multimediale Elemente wie audio, video und canvas unter Verzicht auf Plugins direkt in den Browser zu implementieren und im Rahmen der WHATWG an einem neuen Standard zu arbeiten. Dies beschränkte sich nicht auf HTML - auch JavaScript-APIs wie Web Storage, WebSockets und Drag & drop wurden unter diesem Label eingeführt.

2006 schwenkte das W3C um und beteiligte sich an der Festlegung des neuen Standards. HTML5 liegt seit Oktober 2014 auf den Seiten des W3-Konsortiums als empfohlener Standard vor:

Nach der WHATWG soll HTML5 die letzte Version von HTML sein, zukünftige Änderungen werden in einem living standard ohne weitere Versionsnummern weiterentwickelt. Das W3C arbeitete aber weiter an neuen Zwischenversionen wie HTML5.2 und HTML5.3. Mittlerweile sind diese Arbeiten aber eingestellt und auch das W3C beteiligt sich am HTML living standard ohne Versionsnummern. Aus diesem Grund wird der Begriff HTML5 im SELFWiki nur noch in Abgrenzung zu älteren HTML-Versionen verwendet.

Grundstruktur einer HTML5-Web-Seite

Befehl Erklärung
<!DOCTYPE html> Hiermit wird der Dokumenten-Typ "html" festgelegt
<html lang="en"> Hier wird mit "en" die verwendete Sprache der Webseite beschrieben ("en" = Englisch, oder "de" = Deutsch). Ausserdem beginnt hier für Browser der mit HTML-Code definierte Bereich.
<head> Im sogenannten "Kopf-Bereich" werden für diese Seite alle allgemein gültigen Links, Scripte, der Seiten-Titel usw. festgelegt. Alle hier hinterlegten Vorgaben sind auf der Webseite nicht zu sehen.
<meta charset="UTF-8"> Im sogenannten "Meta-Bereich" wird festgelegt welchen Zeichensatz diese Webseite verwendet. In diesem Beispiel incl. der deutschen Umlaute (Ä,ä,Ö,ö,Ü,ü usw).
<meta http-equiv="X-UA-Compatible" content="IE=edge"> Durch diese Angabe wird die Darstellbarkeit zu Webbrowsern definiert
<meta name="viewport" content="width=device-width, initial-scale=1.0"> Hier wird festgelegt, wie die auszugebende Webseite dargestellt werden soll. In diesem Beispiel nimmt der Browser für den darzustellenden "Content" die Breite des Mediums (Monitor, Tablett, Mobil-Phone), als Ausgabebreite der Webseite. Die dritte ANgabe bezieht sich auf den dar zu stellenden Maßstab. Im Beispiel 1.0 = 100%.
<title>Document</title> In jedem Browser kann ganz oben der "Seitentitel" angezeigt werden. Dieser Titel wird hier definiert
</head> Der "Kopfbereich" endet durch Eingabe dieses "Tag´s". Ab hier beginnt der Bereich der auf dem jeweiligen Medium ausgegeben werden soll.
<body> Der "Content-Bereich" beginnt hier. Alles was ab hier eingegeben wird, kann auf den Ausgabemedien dargestellt werden.
</body> Der "Content-Bereich" endet durch Eingabe dieses "Tag´s". Ab hier endet der Bereich der auf dem jeweiligen Medium ausgegeben werden soll.
</html> Das Web-Seitenende ist erreicht.

HTML - Elemente


Die Typen der HTML-Elemente

Die HTML-Elemente können gepaart oder ungepaart sein. Die gepaarten Elemente haben ein Start-Tag (<>) und ein End-Tag (</>), und der Inhalt des Elements wird zwischen diesen Tags platziert.

Vergessen Sie das Endtag nicht! Obwohl einige HTML-Elemente auch ohne das Endtag korrekt dargestellt werden, verlassen Sie sich darauf nicht. Das kann zu unerwarteten Ergebnissen oder Fehlern führen.

Die leeren Elemente haben keine Endtags. Leere Elemente können im Starttag wie folgt "geschlossen" werden: <br/>.

Die leeren Elemente in HTML sind: .

Der <area> - Tag

Das Tag <area> bestimmt die aktiven Bereiche von Image-Map (Koordinate, Form, Größe usw.), die durch das Tag <map> definiert wird. Durch den Klick auf den aktiven Bereich des Images geschieht eine bestimmte Handlung, zum Beispiel wird eine neue Seite mit mehr Information eröffnet. Das Tag <area> wird immer im Tag <map> platziert. Die aktiven Bereiche können einander überdecken. In diesem Fall wird beim Klicken der Bereich aktiviert, der Im HTML-Code höher liegt. Das Tag <area> wird gepaart verwendet, das Endtag ist obligatorisch.

Der <base> - Tag

Das Tag <base> ist relativ neu in HTML5. Es definiert die Basis-URL für alle relativen URLs im HTML-Dokument. Es bestimmt auch, wie die Links im aktuellen Dokument (im neuen Fenster, im aktuellen Fenster usw.) geöffnet werden müssen. Das Tag <base> wird ungepaart verwendet. Das Endtag ist notwendig. Aber in XHTML ist das Endtag (</base>) obligatorisch. Auf der Seite kann nur ein Tag <base> verwendet werden, und es muss im Tag <head> platziert werden. Sie müssen es so schnell wie möglich einfügen, da seine Wirkung sich von der Stelle aus erstreckt, an der es angegeben ist.

Der <br> - Tag

Das Tag <br> bestimmt die Übertragung der Zeile, d. h. die Stelle, wo die nächste Zeile nach dem Zeilenumbruch beginnt. Im Gegensatz zum Tag <p>, das der Absatz bestimmt, wird vor der Zeile keine Leertaste hinzugefügt. Das Tag <br> wird gepaart benutzt, das Endtag wird nicht verwendet. Aber es ist obligatorisch das Tag <br> im XHTML mit dem Endtag (<br />) zu schließen. Verwenden Sie das Tag <br> nicht anstelle des Tages <p> für die Unterteilung des Textes nach Absätzen.

Der <col> - Tag

Das Tag <col> bestimmt die Eigenschaften einer oder mehreren Spalten in der HTML-Tabelle, die mit dem Tag <table> bestimmt wird. Das Tag <col> wird häufig mit dem Tag <colgroup>, das eine Gruppe mit den Gesamteigenschaften bestimmt, verwendet. Das Tag <col> wird ungepaart verwendet, kein Endtag ist notwendig. In XHTML aber muss das Tag <col> mit dem Endtag (</col>) geschlossen werden. Das Tag <col> wird innerhalb des Tages <table>, vor den Tags <thead>, <tbody>, <tfoot> und <tr>; und nach dem Tag <caption> platziert, wenn es verwendet wird (im Tag <caption> fügen wir den Namen der Tabelle ein).

Der <embed> - Tag

Das Tag <embed> wird als Container für die externen Anwendungen, den multimedialen und interaktiven Inhalt, die der Browser nicht versteht, verwendet. Für ihre korrekte Darstellung muss man externes Plug-in oder spezielle Programme anschließen. Wie der beigefügte Inhalt angezeigt wird, hängt vom Typ der Datei, von den Attributen des Tages <embed> sowie von Plug-ins, die im Browser installiert sind, ab. Obwohl das Tag <embed> von den Browsern unterstützt wurde, war es in HTML 4 nicht eingeschlossen. Fürs Hochladen von Objekten in HTML 4 wird das Tag <object> verwendet. Da nicht alle Browser in sich befindende Information korrekt darstellen, wird für die Unterstützung der großen Anzahl von Browsern sowie die Validität des Dokumentes das Tag <embed> innerhalb des Tages <object> verwendet. In HTML 5 gibt es kein Problem mit der Validität des Dokuments, weil das Tag <embed> bereits ein ofizielles Status in der neuen Version von HTML hat.

Der <hr> - Tag

In HTML 5 wird das Tag <hr> für die thematische Unterteilung des Inhalts auf der Webseite verwendet. In den früheren Versionen von HTML wurde es auf den Webseiten für die Gestaltung von horizontalen Linien, die den Inhalt visuell unterteilen, verwendet. In der neuen Version hat es auch inhaltliche Belastung und bestimmt nicht nur visuelle, sondern auch thematische Unterteilung des Inhalts. Das Tag <hr> wird ungepaart verwendet, kein Endtag ist notwendig. In XHTML ist jedoch das Endtag (<hr/>) obligatorisch. Da das Tag <hr> ein Block-Element ist, werden alle Elemente nach ihm an eine neue Zeile übertragen. Das vom Tag bestimmte Aussehen der horizontalen Linie hängt vom Browsertyp ab.

Der <img> - Tag

Das Tag <img> wird in HTML für die Beschreibung von Bildern verwendet. Das Bild selbst wird im Dokument nicht direkt platziert, der Browser lädt es aus seiner Quelle, die im Tag <img> eingegeben ist. Für die Eingabe der Quelle wird das Attribut src verwendet. Es gibt zwei erforderliche Attribute für das Element <img>: src, das zur Anzeige der Bildquelle verwendet wird, und alt, das einen alternativen Text für das Bild definiert. Um HTML-Bilder klickbar zu machen, sollen Sie das Tag <img> im Tag <a> platzieren. Das Tag <a> wird zum Einfügen von Links verwendet.

Der <input> - Tag

Das Tag <input> wird im Rahmen des Elements <form> für die Bestimmung der Felder für die Informationseingabe seitens der Benutzer verwendet. Der Typ des Feldes (Text, Radiobutton, Checkbox, Feld für die Passworteingabe usw.) wird durch das Attribut type definiert. Um den Text mit einem bestimmten Element zu verbinden, wird das Tag <label>, das für ihn die Textmarkierung definiert, verwendet. Das Tag <input> wird ungepaart verwendet, kein Endtag ist notwendig. Das Tag enthält keinen Textinhalt, es enthält nur Attribute.

Der <keygen> - Tag

Das Tag <keygen> ist für die Generierung eines Schlüsselpaares (öffentlich und privat) verantwortlich, das zum Verschlüsseln und Entschlüsseln von Daten, die beim Senden eines HTML-Formulars an den Server gesendet werden, verwendet wird. Der öffentliche Schlüssel wird zusammen mit den übermittelten Formulardaten an den Server gesendet, und der private Schlüssel wird auf dem lokalen Gerät des Benutzers gespeichert. Das Tag wird auch zum Erstellen und Verifizieren der digitalen Signatur verwendet.

Der <link> - Tag

Das Tag <link> verbindet das HTML-Dokument mit externer Quelle. In der Regel wird es für die Einschaltung des externen Files, wo CSS-Stile und Schrifte eingegeben werden, verwendet. Das Tag <link> wird ungepaart verwendet, es enthält nur Attribute und wird immer innerhalb des Tages <head> (in der Regel vor dem Endtag </head>) platziert, dabei kann es mehrmals verwendet werden.

Der <meta> - Tag

Das Tag <meta> enthält Metadaten, die dem Browser und der Suchmaschine über die Einzelheiten des HTML-Dokuments (Codierung des Dokuments, Angaben des Autors, Schlüsselwörter, Beschreibung von Webseiten usw.) informieren. Die Metadaten werden auf der Webseite nicht angezeigt. Das Tag <meta> wird im Tag <head> platziert. In einem Dokument kann man einige Tags <meta> verwenden.

Der "param" - Tag

Das Tag <param> definiert die Parameter von Java-Applets oder Multimedia-Inhalt, die in den Elementen <applet> oder <object> platziert werden. Dieses Tag erlaubt, die Charakteristik von Objekten direkt im Code des HTML-Dokuments zu ändern. Das Tag <param> bestimmt auch die Parameter für die Abspielung von geeigneten Audio und Videodateien, die durch die neuen HTML5-Tags <audio> und <video> bestimmt werden. Es kann mehr als ein Tag <param> verwendet werden, für jedes von denen wird das Paar “Name-Wert” durch die Attribute name und value festgelegt.

Der "source" - Tag

Das Tag <source> bestimmt einige multimediale Dateien in verschiedenen Formaten wie Audio und Video. Es ist für die Eigenschaft Cross-Browser notwendig: Der Browser kann aus zugänglichen Varianten das Format, das er unterstützt und ohne Probleme Audio- oder Videodateien abspielt, wählen. In einem Dokument kann das Element <source> mehrmals für die Eingabe von alternativen Audio- und Videodateinen und die Eingabe von einigen Formaten verwendet werden.

Beispiel:

<track src="...">

Der "track" - Tag

Das Tag <track> definiert eine externe Textspur für die Elemente <audio> und <video>, die gewöhnlich Untertitel in verschiedenen Sprachen, Kommentare, Titel usw. enthält.

Beispiel:

<track src="...">

Der "wbr" - Tag

HTML Tag <wbr> weist den Browser darauf hin, dass bei Bedarf einen Zeilenumbruch erlaubt ist. Dieses Element unterscheidet sich vom Tag <br> dadurch, dass beim Gebrauch des Tages <br> der Browser unbedingt einen Zeilenumbruch macht, beim Gebrauch des Tages <wbr> der Browser zuerst den Inhalt des Elements analysiert und nur dann einen Zeilenumbruch macht, nur wenn es notwendig ist (zum Beispiel, zu langes Wort oder lange URL-Adresse).
