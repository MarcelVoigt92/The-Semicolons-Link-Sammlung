Git / Git-Hub

GitHub ist ein webbasierter Filehosting-Service für die Softwareentwicklung und das Teilen von Softwarecode. Der Service bietet kostenfreie öffentliche Repositories und Collaboration-Funktionen für die Entwicklung von Open-Source-Software . Gleichzeitig können kleine und mittelständische Unternehmen (KMU) gegen eine Gebühr private Repositories und Collaboration-Feature mieten.

GitHub ermöglicht Anwendern in seinen öffentlichen Repositories Software zu verändern, anzupassen und weiterzuentwickeln. Jedes Projekt wird als Fork (Abspaltung) bezeichnet. Entwickler können auf GitHub als Team oder allein arbeiten. Öffentliche Nutzer erstellen dafür Profile, welche ihre Respositories und öffentlichen Aktivitäten zeigen. Zudem helfen die Profile Entwicklern beim Finden von Projekten.

GitHub wurde 2008 ins Leben gerufen und basiert auf Git, einem Open-Source-Code-Management-System, das von Linus Torvald entwickelt wurde, um Software-Builds schneller erstellen zu können. Nachdem 2011 eine Sicherheitslücke innerhalb des Standard- Linux-Kernels auf kernel.org auftauchte, veröffentlichte Torvald Release 3.0 des Linux-Kernels auf GitHub.

Weitere Gruppen und Organisationen, die Projekte in öffentlichen Repositories auf GitHub veröffentlicht haben oder veröffentlichen, sind: Red Hat mit der JBoss-Software, Twitter, Microsoft Azure , das Open-Compute-Projekt und Mozilla.

Die wichtigsten Git-Befehle

Mit git --version überprüft ihr, welche git-Version installiert ist.

Mit git config --global user.name "USERNAME" legt ihr euren globalen Nutzernamen fest, welcher der gleiche wie in Git-Hub sein sollte damit ihr euch korrekt beim Code pushen identifiziert.

Mit git config --global user.email "e-mail Adresse" legt ihr die dazugehörtige E-Mail Adresse an (auch diese sollte die selbe wie bei Git-Hub sein).

Mit git clone "SSH-Key oder HTML-Key" könnt ihr ein Repositorie aus Git-Hub klonen und so lokal auf eurem Laptop speichern.

Mit git status könnt ihr den aktuellen Status des Repositories einsehen.

Mit git add könnt ihr von euch vorgenommene Änderungen zu dem Repositorie lokal hinzufügen und für den Commit vorbereiten.

Mit git commit -m „Commit-Nachricht“ staged ihr die bereits geaddeten Änderungen.

Mit git push REMOTE BRANCHNAME fügt ihr die gestageten Änderungen dem remote Repositorie hinzu. REMOTE ist dabei der Name des entfernten Servers. Wenn Du ein Projekt klonst, dann ist der von git vergebene Standardname origin. Zudem kannst du angeben, in welchen Branch das Projekt hinzugefügt werden soll.

Mit git push origin master werden die Änderungen dem Master Branch hinzugefügt.

Mit git fetch könnt ihr Änderungen aus dem remote Repositorie in das lokale Repositorie zu holen. Der git fetch Befehl wird in diesem Video genauer erklärt: Click Me

Mit git branch -a lasst ihr euch alle angelegten Branches anzeigen.

Mit git checkout BRANCH könnt ihr zwischen den Branches hin und her springen.

Mit git pull kannst du Änderungen aus einem entfernten Repository abholen und mit deinem Workspace, also den Dateien, an denen du derzeit arbeitest, synchronisieren. Du kannst dabei den Pfad zu einem Repository angeben sowie den Branch, aus dem du Änderungen abholen möchtest. Kurz gesagt: git pull = git fetch + git merge.

Mit git checkout -b "Branchname" erstellt einen neuen Branch und wechselt direkt in diesen.

Mit git checkout "Branchname" springt ihr zwischen den vorhandenen Branches hin und her.
